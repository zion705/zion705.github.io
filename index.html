          01 / PRODUCT INSIGHT

          问题不是缺少行情工具，
          而是判断很难被持续练习。

          “没人会关心陌生人的收益率排行榜，
          但所有人都会关心朋友今天又亏了多少。”
          持续动力

          传统模拟盘像一份作业

          没有熟人关系和局内悬念，新手很难跨过一次性体验，形成连续练习。

          即时反馈

          盈亏数字不能解释判断

          用户知道自己赚或亏，却不知道是信息、时机、风险偏好还是从众造成结果。

          经验沉淀

          交易结束，经验也随之消失

          缺少被组织的复盘语言，用户很难把一次输赢迁移到下一次决策。

          02 / CORE LOOP

          让每一次交易，进入下一轮判断。

          我将分散的投资、社交和 AI 能力组织为一个可重复闭环。 点击图中的节点，查看每一步承担的产品任务。

          03 / PRODUCT SCOPE

          功能取舍不是做少，
          而是保护核心体验。

          A

          熟人房间，而不是陌生人排行榜

          把竞争建立在已有关系上，让亏损、质疑和反转天然具备传播与讨论价值。

          关系张力 > 排名刺激
          B

          干扰信息，而不是操纵股价

          卧底只能影响信息环境，不能改变价格，保留用户判断与真实市场逻辑之间的联系。

          信息博弈 > 数值作弊
          C

          AI 教练，而不是自动投顾

          AI 提供判断依据、行为归因与复盘，不替用户做决定，也不包装为真实投资建议。

          能力增强 > 决策替代
          04 / INFORMATION ARCHITECTURE

          白天做判断，晚上解释判断。

          信息架构按用户心智切成两个场域：交易所承接数据与行动， 小酒馆承接表达、质疑、战报和投票。

          05 / AGENT DESIGN

          AI 不只回答问题，
          它要读懂这一局。

          我和团队共同参与 Agent 方案设计，我重点推动能力进入具体链路： 何时出现、读取什么上下文、输出什么，以及如何守住风险边界。

          CONTEXT 当前股票 持仓与交易 新闻事件 玩家言论
          →
          AGENT ROLES 投资导师解释标的与风险 行为分析师识别节奏与异常 复盘官组织证据与战报
          →
          OUTPUT 判断依据 行动建议 行为归因 AI 战报
          评测关注
          问答相关性

          回答是否使用当前标的、持仓和新闻，而不是输出通用话术。

          行为归因

          能否把异常交易与时点、信息链和玩家表达建立可解释联系。

          风险边界

          明确这是游戏模拟，不伪装为可靠的真实投资建议。

          07 / RESULT & NEXT

          奖励验证了创意吸引力，
          但不是用户价值的终点。

          已验证 核心闭环可运行

          从选股、交易、AI 咨询，到战报、投票和终局均可在原型中完成。

          外部信号 清华黑客松最佳创意奖

          证明概念在短时间内具备可理解性和差异化，但不替代长期用户验证。

          仍待验证 是否真的改善判断学习

          需要观察五日留存、战报阅读、投票参与，以及复盘后决策是否发生变化。

          邀请接受率 首轮交易完成率 战报阅读率 投票参与率 次日回流率 战报分享率
